Crypto Data Pipeline

Extracting and Visualizing Cryptocurrency Market Data with PostgreSQL and Looker Studio

Project Overview

This project connects a backend server to the CoinGecko API to extract real-time data on the most important cryptocurrencies. The data is stored in a PostgreSQL database and visualized through a Looker Studio dashboard.

Data Extraction and Storage

  • The backend server queries CoinGecko API for key cryptocurrencies.
  • Extracted data includes minimum price, maximum price, current price, and average price over time.
  • Data is inserted and updated in a PostgreSQL database to maintain historical records.

Dashboard Visualization

The Looker Studio dashboard provides two main views:

  • A table showing the minimum, maximum, current, and average prices of each cryptocurrency.
  • A historical price chart visualizing price trends over time.
  • A separate page displays a volatility chart comparing the price fluctuations of each cryptocurrency.

Technologies Used

  • Backend: Python Flask server for API integration and data handling.
  • Database: PostgreSQL for robust data storage and historical tracking.
  • Visualization: Looker Studio for interactive dashboards and data presentation.
  • API: CoinGecko public API for reliable cryptocurrency market data.

How It Works

  1. The server sends requests to CoinGecko API.
  2. Retrieved JSON data is parsed and transformed as needed.
  3. Processed data is saved in PostgreSQL, ensuring up-to-date and historical accuracy.
  4. Looker Studio connects to the database to fetch data and refresh dashboard visualizations automatically.

Benefits

  • Automated data pipeline reduces manual data collection and errors.
  • Historical tracking enables trend analysis and volatility assessment.
  • Interactive dashboards provide easy-to-understand insights for decision-making.
  • Scalable architecture allows adding more cryptocurrencies or metrics as needed.
